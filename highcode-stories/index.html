Nº 02/The process

Three movements,
quietly executed.

The submission is read in the browser, parsed on a private pipeline, and filed in the project of record. No copy-paste, no manual transcription.

I.

Upload

Drop a PDF, Word file, or Markdown specification. DOCX is converted to text in your browser before it leaves the page.

Submission
II.

Extract

An n8n pipeline calls a language model to read the document, identify epics and stories, and write summaries worth shipping.

Reading
III.

Ship

Issues land in the Jira RN project, linked back to the source document. The archive records each run with deep links to every issue created.
